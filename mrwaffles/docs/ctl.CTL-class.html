canonic(self, noand=False)

source code 

returns a new equivalent formula with only !, or, and, EX, EU, EG and EGi

if noand is True, 'and's will be turned in 'or's

decompose(self)

source code 

if formula := atomic -> 'atomic'

:= !formula -> '!', formula

:= or f1 f2 -> 'or', f1, f2

...

match(self, pattern)

source code 

pattern can be a string or a CTL instance

pattern can be a string of the form 'term', 'monoop term' or 'binop term1 term2'

the '_' token can match any single token (a term, an atomic or an operator)

two formulas will match if they have the same arity, the same operator and their subterms match

match_sub(self, pattern, subs=None, debug=0)

source code 

same as match() but returns a tuple (boolean, dictionary)

the dictionary maps free variables to the terms they match

simplify(self)

source code 

returns a new equivalent formula

e.g. !!pouet -> pouet, !true -> false, or true f -> f

only minimal operators are simplified, so call canonic() first

subformulas(self)

source code 

iterator over subformulas of this formula

example: f = And(f1, f2)

f.subformulas() will yield f1 and f2

the subformulas are CTL instances
